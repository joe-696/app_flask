================================================================================
           MANUAL DE PRUEBAS - SISTEMA DE GESTIÓN DE PEDIDOS DE RESTAURANTE
================================================================================

INFORMACIÓN GENERAL:
- Sistema: Aplicación web Flask para gestión de pedidos de restaurante
- URL de acceso: http://127.0.0.1:5000
- Base de datos: SQLite (se crea automáticamente)
- Navegador recomendado: Chrome, Firefox, Edge

================================================================================
                            PREPARACIÓN PARA PRUEBAS
================================================================================

1. INICIAR LA APLICACIÓN:
   - Abrir terminal en la carpeta del proyecto
   - Ejecutar: python app.py
   - Verificar mensaje: "Servidor iniciado en http://127.0.0.1:5000"
   - Abrir navegador y ir a: http://127.0.0.1:5000

2. DATOS DE PRUEBA:
   La aplicación incluye datos de muestra:
   - 5 productos predefinidos (Pizza Margherita, Hamburguesa, Ensalada César, etc.)
   - 2 pedidos de ejemplo
   - Diferentes estados: pendiente, en_preparacion, listo, entregado

MÓDULO 1: NAVEGACIÓN PRINCIPAL
------------------------------
TC01 - Página de Inicio
□ Abrir http://127.0.0.1:5000
□ Verificar que se muestra la página de bienvenida
□ Verificar que aparece el título "Gestión de Pedidos - Restaurante"
□ Verificar que se muestran las estadísticas del día
□ Verificar que aparece la sección "Pedidos Recientes"

□ Verificar que la barra de navegación contiene:
  - Logo/Título del sistema
  - Enlace "Inicio"
  - Enlace "Pedidos"
  - Enlace "Productos"
□ Hacer clic en cada enlace y verificar que funciona correctamente

MÓDULO 2: GESTIÓN DE PRODUCTOS
-------------------------------
TC03 - Listar Productos
□ Ir a "Productos" desde el menú

□ Hacer clic en "Guardar"
□ Verificar mensaje de confirmación
□ Verificar que el producto aparece en la lista

TC05 - Validaciones en Formulario de Producto
□ Intentar crear producto con campos vacíos
□ Verificar que se muestran mensajes de error
□ Intentar ingresar precio negativo
□ Intentar ingresar texto en campo precio
□ Verificar que las validaciones funcionan correctamente

TC06 - Editar Producto
□ En la lista de productos, hacer clic en "Editar" de un producto
□ Modificar algunos campos
□ Guardar cambios
□ Verificar que los cambios se reflejan en la lista

TC07 - Eliminar Producto
□ En la lista de productos, hacer clic en "Eliminar"
□ Confirmar eliminación
□ Verificar que el producto ya no aparece en la lista

□ En la lista de pedidos, hacer clic en "Ver" de un pedido

CRITERIOS DE ACEPTACIÓN
-----------------------
El sistema pasa las pruebas si:
✓ Todas las funcionalidades básicas funcionan sin errores
✓ Las validaciones previenen datos incorrectos
✓ La navegación es intuitiva y fluida
✓ Los datos se guardan y recuperan correctamente
✓ La interfaz es responsive y profesional
✓ Los mensajes de error/éxito son claros y útiles

================================================================================

REPORTE DE DEFECTOS
-------------------
Para cada error encontrado, documentar:
1. ID del caso de prueba donde ocurre
2. Descripción del error
3. Pasos para reproducirlo
4. Resultado esperado vs. resultado actual
5. Severidad (Alta/Media/Baja)

Ejemplo:
TC09 - Error en filtro "En preparación"
- El filtro no muestra pedidos cuando se selecciona "En preparación"
- Pasos: Ir a Pedidos > Seleccionar filtro "En preparación"
- Esperado: Mostrar pedidos en ese estado
- Actual: Página vacía
- Severidad: Alta

================================================================================

NOTAS ADICIONALES
-----------------
- Si la aplicación no inicia, verificar que Python y las dependencias estén instaladas
- Para reiniciar datos de prueba, eliminar archivo "restaurante.db" y reiniciar la aplicación
- Los datos se guardan automáticamente, no se requiere "guardar" manual
- El sistema funciona completamente offline (no requiere internet)
